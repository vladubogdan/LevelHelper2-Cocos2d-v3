Release Notes

  • Support for Cocos2d 3.3

After Update Compilation Help

Because of changes in the Cocos2d 3.3 library files structure, you will need to change your Xcode project "Header Search Paths".

1. Select you [ProjectName]+Box2d+iOS target
2. Navigate to Build Settings
3. Search for "Header Search Paths"
4. Double click and add the following paths
  • "$(SRCROOT)/cocos2d/cocos2d" - "recursive"
  • "$(SRCROOT)/cocos2d/external/Chipmunk/objectivec/include" - "non-recursive"
  • "$(SRCROOT)/cocos2d/external/Chipmunk/chipmunk/include" - "non-recursive"
  • "$(SRCROOT)/cocos2d/cocos2d-ui" - "recursive"
  • "$(SRCROOT)/cocos2d/external/ObjectAL" - "recursive"
  • "$(SRCROOT)/Box2D" - "recursive"
  • "$(SRCROOT)/LevelHelper2-API/LevelHelper2-API" - "recursive"
5. Remove the old paths, but make sure you keep your own paths if any.
